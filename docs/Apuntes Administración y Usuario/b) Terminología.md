# Terminología Común en WordPress

WordPress tiene una terminología específica que puede resultar confusa para los principiantes. Este glosario incluye términos clave para ayudarte a comprender mejor el funcionamiento del CMS y su ecosistema.

---

## 1. **Términos Relacionados con el Núcleo de WordPress**

1. **WordPress Core:**
   - El núcleo o "core" de WordPress es el software base que proporciona todas las funcionalidades esenciales del CMS.

2. **Admin Dashboard:**
   - Es el panel de administración de WordPress desde donde gestionas todo el contenido y la configuración del sitio.

3. **Posts y Pages:**
   - **Posts (Entradas):** Contenido cronológico, como publicaciones de blogs.
   - **Pages (Páginas):** Contenido estático, como "Sobre Nosotros" o "Contacto".

4. **Permalinks:**
   - Enlaces permanentes o URLs utilizadas para acceder a las entradas, páginas y otros contenidos del sitio.

---

## 2. **Términos Relacionados con la Apariencia**

1. **Themes (Temas):**
   - Plantillas que determinan el diseño y la presentación visual del sitio.

2. **Widgets:**
   - Elementos que puedes añadir a áreas específicas del sitio (como barras laterales o pies de página) para mostrar contenido adicional.

3. **Customizer:**
   - Herramienta integrada que permite previsualizar y ajustar la apariencia del sitio en tiempo real.

4. **Templates:**
   - Archivos específicos dentro de un tema que definen cómo se muestra cierto contenido (por ejemplo, una página de archivo o una entrada individual).

---

## 3. **Términos Relacionados con la Funcionalidad**

1. **Plugins:**
   - Extensiones que añaden funcionalidades adicionales a WordPress, como formularios de contacto, optimización SEO o integración de redes sociales.

2. **Shortcodes:**
   - Códigos cortos que se utilizan para insertar funciones dinámicas en el contenido (por ejemplo, `[gallery]`).

3. **Taxonomies:**
   - Sistemas para organizar contenido, como categorías y etiquetas.

4. **Custom Post Types:**
   - Tipos de contenido personalizados que se pueden crear además de las entradas y páginas estándar (por ejemplo, "Proyectos" o "Testimonios").

---

## 4. **Términos Relacionados con el Backend**

1. **Database (Base de Datos):**
   - WordPress utiliza una base de datos MySQL o MariaDB para almacenar contenido, configuraciones y usuarios.

2. **Hooks (Acciones y Filtros):**
   - **Acciones:** Eventos específicos a los que puedes conectar funciones personalizadas.
   - **Filtros:** Permiten modificar datos antes de que sean procesados o mostrados.

3. **WP-CLI:**
   - Herramienta de línea de comandos para administrar WordPress de manera eficiente.

4. **Multisite:**
   - Una funcionalidad que permite gestionar múltiples sitios desde una sola instalación de WordPress.

5. **REST API:**  
    Interfaz que permite que aplicaciones externas (como apps móviles o frontends headless) se comuniquen con WordPress.
    
6. **Site Health:**  
    Panel integrado que muestra el estado de salud del sitio, incluyendo recomendaciones de rendimiento y seguridad.
    
7. **Application Passwords:**  
    Contraseñas generadas para aplicaciones externas que se conectan a WordPress de forma segura.

---

## 5. **Términos Relacionados con la Seguridad y Mantenimiento**

1. **Roles de Usuario:**
   - Niveles de permisos asignados a usuarios, como Administrador, Editor, Autor, Colaborador y Suscriptor.

2. **Backups:**
   - Copias de seguridad del contenido y la configuración del sitio.

3. **Updates:**
   - Actualizaciones del núcleo de WordPress, temas y plugins.

4. **SSL/TLS:**
   - Protocolos que encriptan la conexión entre el servidor y los visitantes del sitio para mayor seguridad.

## 6. **Términos Relacionados con el Editor de Bloques y FSE**

#### 1. **Editor de Bloques (Gutenberg)**


1. **Block Editor (Gutenberg)**
   - Editor visual basado en bloques que reemplazó al editor clásico. Permite construir páginas y entradas con bloques reutilizables y patrones de diseño.

2. **Blocks**
   - Elementos individuales que componen el contenido en el editor de bloques. Pueden ser nativos, de plugins o personalizados, por ejemplo párrafo, imágen, botón, etc.)

3.  **Patterns (Patrones de Bloques)**
   - Diseños predefinidos de bloques que se pueden insertar y personalizar.

4.  **Full Site Editing (FSE)
   - Funcionalidad que permite editar todo el sitio (incluyendo cabecera, pie y plantillas) con el editor de bloques.
   
5.  **Global Styles**
   - Herramienta que permite definir estilos globales (colores, fuentes, márgenes) desde el editor de bloques.

6.  **Template Parts**
   - Fragmentos de plantilla reutilizables (como cabecera o pie) en temas que soportan Full Site Editing.
   
7.  **Reusable Blocks**
   - Bloques personalizados que se guardan para ser reutilizados en múltiples entradas o páginas.
