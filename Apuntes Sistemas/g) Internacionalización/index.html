    Modelo de Internacionalización en WordPress¶

    WordPress tiene un sistema robusto de internacionalización (i18n) y localización (l10n) que permite traducir y adaptar un sitio web o un plugin a diferentes idiomas y culturas. Este modelo facilita la creación de sitios web multilingües y accesibles globalmente.

    1. Conceptos Clave¶

    1. Internacionalización (i18n):
    2. Proceso de preparar el código para que pueda traducirse fácilmente.

    3. Incluye el uso de funciones específicas para envolver textos que se mostrarán en diferentes idiomas.

    4. Localización (l10n):

    5. Proceso de traducir y adaptar el contenido a un idioma y cultura específicos.

    6. Incluye archivos de traducción con extensiones .mo y .po.

    7. Idiomas en WordPress:

    8. WordPress soporta más de 70 idiomas gracias a su comunidad de traducción.
    9. Los idiomas del sitio pueden configurarse desde el panel de administración.

    2. Configuración del Idioma del Sitio¶

    1. Ve a Ajustes > Generales en el panel de administración.
    2. En "Idioma del sitio", selecciona el idioma deseado del menú desplegable.
    3. Guarda los cambios.
    4. WordPress descargará automáticamente los archivos de idioma correspondientes si están disponibles.

    3. Archivos de Traducción en WordPress¶

    Estructura de Archivos¶

    1. Archivos .po (Portable Object):
    2. Contienen las cadenas de texto en su idioma original y las traducciones.

    3. Se editan con herramientas como Poedit.

    4. Archivos .mo (Machine Object):

    5. Versión compilada del archivo .po que WordPress utiliza para mostrar traducciones.

    6. Ubicación de Archivos de Idioma:

    7. Idiomas del núcleo de WordPress: 
      /wp-content/languages
    8. Idiomas de temas: 
      /wp-content/themes/tu-tema/languages
/wp-content/themes/languages/*

    9. Idiomas de plugins: 

      /wp-content/plugins/tu-plugin/languages
/wp-content/plugins/languages/*

    10. Archivos json:

    11. Desde hace mucho tiempo los plugins y temas pueden incluir traducciones para el editor de bloques en archivos .json ubicados en /languages/

    4. Funciones de Internacionalización en WordPress¶

    1. __() y _e()
    2. __() devuelve una cadena traducida.

    3. _e() imprime directamente una cadena traducida. 

      echo __( 'Hello, World!', 'text-domain' );
_e( 'Welcome to my site.', 'text-domain' );

    4. _x() y ex()

    5. Proporcionan contexto adicional para cadenas ambiguas. 

      echo _x( 'Post', 'noun', 'text-domain' );

    6. ngettext()

    7. Maneja traducciones para singular y plural. 

      echo sprintf( _n( '%s comment', '%s comments', $count, 'text-domain' ), $count );

    8. load_textdomain()

    9. Carga archivos de traducción manualmente. 
      load_textdomain( 'text-domain', WP_LANG_DIR . '/plugins/my-plugin-es_ES.mo' );

    5. Traducción de Temas y Plugins¶

    1. Añadir un Text Domain
    2. Define un "text domain" en el encabezado del archivo principal del tema o plugin. 
      /*
 * Plugin Name: Mi Plugin
 * Text Domain: mi-plugin
 */

    Ejemplo: para el tema Astra podemos encontrar la identificación del dominio en la ruta /themes/astra/inc/class-astra-after-setup-theme.php. Podemos usar el comando grep -Ri "Text Domain" . para buscar en generar los text domains existentes.

    1. Preparar el Código para Traducción
    2. Usa funciones de internacionalización para envolver cadenas de texto.

    3. Asegúrate de que el "text domain" coincida con el definido en el encabezado.

    4. Generar Archivos .pot (Portable Object Template)

    5. Usa herramientas como Poedit o plugins como Loco Translate para extraer las cadenas traducibles.

    6. Subir Archivos de Traducción

    7. Coloca los archivos .mo y .po en las carpetas correspondientes.

    6. Sitios Multilingües¶

    WordPress no tiene soporte multilingüe nativo para gestionar varios idiomas en un mismo sitio. Aunque se pueden crear versiones separadas de contenido manualmente, esto requiere una gran cantidad de trabajo y no es eficiente ni práctico para sitios más grandes o con actualizaciones frecuentes. Por ello, es necesario utilizar plugins multilingües que gestionen automáticamente el contenido en diferentes idiomas y simplifiquen la administración del sitio.

    ¿Por qué hacen falta plugins para sitios multilingües?¶

    1. Gestión Centralizada de Idiomas:

    2. Los plugins permiten configurar y administrar varios idiomas desde un solo panel, evitando la necesidad de crear múltiples sitios o páginas duplicadas manualmente.

    3. Traducción de Contenido:

    4. Facilitan la traducción de entradas, páginas, menús, widgets y otros elementos, proporcionando interfaces dedicadas para realizar esta tarea.

    5. SEO Multilingüe:

    6. Aseguran que cada idioma tenga una estructura de URL adecuada (por ejemplo, example.com/en/ para inglés y example.com/es/ para español), mejorando el SEO y la visibilidad en motores de búsqueda.

    7. Sincronización de Contenido:

    8. Los plugins gestionan la relación entre el contenido en diferentes idiomas, asegurando que las actualizaciones en un idioma puedan replicarse fácilmente en los demás.

    9. Widgets y Funciones Especializadas:

    10. Ofrecen herramientas como selectores de idioma, traducción automática y opciones avanzadas para personalizar la experiencia del usuario.

    11. Compatibilidad con Otros Plugins y Temas:

    12. Los plugins multilingües garantizan que el contenido traducido sea compatible con otros plugins y temas instalados, evitando conflictos o problemas de funcionalidad.

    Plugins Multilingües Populares¶

    1. Polylang:
    2. Configura varios idiomas en tu sitio.

    3. Traduce entradas, páginas, menús y widgets.

    4. WPML (WordPress Multilingual Plugin):

    5. Solución completa para sitios multilingües con opciones avanzadas.

    6. Weglot:

    7. Servicio basado en la nube para traducción automática y manual.

    8. TranslatePress:

    9. Permite traducir el contenido directamente desde la interfaz del sitio.

    7. Mejores Prácticas para Internacionalización¶

    1. Usa Funciones de WordPress:

    2. Garantiza compatibilidad futura y consistencia.

    3. Evita Cadenas de Texto Fijas:

    4. No incluyas texto directamente en HTML o PHP sin envolverlo en funciones de traducción.

    5. Usa Text Domains Únicos:

    6. Asegúrate de que cada tema o plugin tenga su propio dominio de texto.

    7. Prueba con Idiomas Diferentes:

    8. Cambia el idioma del sitio y verifica que todas las cadenas se traduzcan correctamente.
